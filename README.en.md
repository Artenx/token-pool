# Token Pool

> LLM API Pool Manager — Multi-endpoint load balancing & token quota management

[中文文档](./README.md)

## Features

- **Multi-endpoint pools** - Group endpoints into pools with passthrough or mapping model name modes
- **Load balancing** - Round Robin, Failover, and Random algorithms with automatic retry on failure
- **Token quota management** - Supports unlimited, manual reset, and daily auto-reset
- **Custom routing** - Configure external API prefixes with independent authentication keys
- **Web dashboard** - Visual management of endpoints, pools, APIs, and system settings

## Quick Start

### Option 1: Build from source

#### Prerequisites

- Linux
- Rust toolchain (1.75+)

#### Build

```bash
cd token-pool
cargo build --release
```

#### Deploy

```bash
# Create deployment directory
mkdir -p /opt/token-pool

# Copy files
cp target/release/token-pool /opt/token-pool/
cp -r static /opt/token-pool/

# Install systemd service
cp token-pool.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable token-pool
systemctl start token-pool
```

#### Verify

```bash
# Check service status
systemctl status token-pool

# Health check
curl http://localhost:8080/health
```

---

### Option 2: Docker

#### Prerequisites

- Docker
- Docker Compose (optional)

#### Docker Compose (recommended)

```bash
cd token-pool

# Create data directory
mkdir -p data

# Build and start
docker-compose up -d

# View logs
docker-compose logs -f

# Stop
docker-compose down
```

#### Docker CLI

```bash
# Build image
docker build -t token-pool .

# Run container
docker run -d \
  --name token-pool \
  -p 8080:8080 \
  -v $(pwd)/data:/app/data \
  --restart unless-stopped \
  token-pool
```

#### Verify

```bash
# Check container status
docker ps | grep token-pool

# Health check
curl http://localhost:8080/health
```

## Endpoints

| Item | URL |
|------|-----|
| Dashboard | `http://your-ip:8080/admin/` |
| Default password | `admin123` |
| API endpoint | `http://your-ip:8080/{prefix}/chat/completions` |

## Usage

```bash
# Call API via configured prefix
curl http://your-ip:8080/your-prefix/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-api-key" \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello"}],
    "max_tokens": 50
  }'
```

## Configuration

Config file is located at `/opt/token-pool/config.toml`, auto-generated on first startup.
